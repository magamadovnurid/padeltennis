# Padel Next Point

Локальный mobile-first PWA-прототип для падел-платформы:
- installable web app;
- главная, матчи, турниры, рейтинг, профиль;
- локальная сессия по email;
- dev-заглушка под вход через MAX.

## UI Guide

Базовые правила интерфейса для проекта:

- Только короткие заголовки, метки, числа и действия.
- Не использовать длинные объясняющие абзацы на экранах.
- Каждый экран должен читаться за 2-3 секунды.
- Вторичный текст должен оставаться контрастным и читаемым на телефоне.
- Текст в нижнем меню обязан быть виден всегда, не только в активном состоянии.
- На тёмных карточках не использовать слишком прозрачный белый текст.
- Кнопки должны иметь явный контраст фона и текста.
- Главная страница начинается сразу с оранжевого блока `Играй сегодня`.
- Верхний header на главной не показывается.
- Install popup должен быть коротким: 1 действие или 3 коротких шага максимум.

## Home Screen

Главная страница строится так:

1. Оранжевый блок `Играй сегодня`.
2. Короткие счётчики.
3. Быстрые действия.
4. Карточки турниров.
5. Карточки матчей.

## Контраст

Обязательные правила по видимости:

- Неактивные пункты нижнего меню не делать полупрозрачными до состояния плохой читаемости.
- Если фон тёмный, текст должен быть светлым и с высоким контрастом.
- Если фон светлый, вторичный текст не должен уходить в слишком бледный серый.
- Любой текст, который не читается без выделения, считается дефектом UI.

## Запуск

```bash
npm install
npm run dev
```

Открыть: `http://localhost:3000`

## Что уже есть

- `Next.js 16` + `App Router`
- PWA manifest
- service worker
- app-like mobile shell
- локальные мок-данные для матчей, турниров и рейтинга

## Что делать следующим шагом

1. Подключить локальную БД (`SQLite + Prisma`).
2. Вынести сессию из `localStorage` в серверную auth-логику.
3. Подключить реальный провайдер MAX после получения понятного API/credential flow.
4. Добавить создание матчей, подтверждение результатов и admin flow.
